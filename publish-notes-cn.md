# Spreadsheet Cleanup Demo 发布说明

## 目的

这个目录是 Spreadsheet Cleanup + Mini Dashboard 服务的 GitHub demo 展示页。它用于 First Dollar Challenge 的获客验证，让潜在客户快速理解：

“我发一个混乱 Excel/CSV，你会返回清洗后的表格和一个简单 Dashboard。”

## 如何发布到 GitHub

1. 在 GitHub 创建一个新 repository。
2. 将 `04-Products/spreadsheet-cleanup-demo/` 目录中的文件上传到 repository 根目录。
3. 确认 repository 至少包含：
   - `README.md`
   - `messy-sales-data.csv`
   - `cleaned-sales-data.csv`
   - `mini-dashboard.md`
   - `link-share-copy.md`
   - `publish-notes-cn.md`
4. 发布后打开 repository 首页，确认 README 能正常显示表格和链接。
5. 复制 GitHub repository URL，用于后续外联。

## 发布后如何把链接用于外联

建议使用方式：

1. 只发给高相关潜在客户，不群发。
2. 先写一句个性化 context，再附上 demo 链接。
3. 明确说明这是 simulated demo，不是真实客户案例。
4. 不要说“我已经帮客户提升收入”。
5. 询问对方是否有一个类似的 messy spreadsheet。
6. 对方感兴趣后，再发送 scope 和付款确认。

示例中文内部流程：

1. 选择一个 ecommerce seller、freelancer 或 SaaS founder。
2. 查看对方是否可能使用 spreadsheet 管理 orders、leads、clients 或 waitlist。
3. 使用 `link-share-copy.md` 中的英文文案做个性化改写。
4. 发送后记录到 Experiment 001 的 outreach log。
5. 如果有回复，优先确认 scope，而不是马上索要文件。

## 注意事项

- 不要声称这是客户案例，只能说 simulated demo。
- 不要编造客户、收入、下载量、访问量或转化率。
- 不要承诺提升 revenue、profit、conversion 或 sales。
- 不要在禁止 self-promotion 的社区发布。
- 不要批量私信。
- 不要让客户发送敏感个人数据、payment card info、bank details、tax IDs、medical/legal records 或 regulated data。
